# Project Memory

## Working Rules From User
1. Security is the top priority.
2. No network connections of any kind, and no feature that could expose the machine through connectivity surfaces.
3. Offline-only. App must run only when explicitly opened by the user, must not auto-start with Windows, must not run in the background, and must not track the user.
4. Code should stay compact, clean, readable, and lightweight.
5. Remove unused code. Do not keep dead code or surplus features.

## Product Decisions Confirmed By User
- Platform target: Windows 10/11 only.
- Keep all core screenshot + annotation features.
- No system tray.
- No global hotkeys.
- Closing the editor exits the app completely.
- Keep local clipboard copy via `Ctrl+C`.
- Keep save via `Ctrl+S`.
- Default save location should prefer Desktop.
- Deliver both source-run and `.exe` build paths later, but do not build while still actively developing unless the user explicitly asks.
- Remove anything not actually used.

## Current App Flow
1. `main.py` creates `QApplication` and opens capture immediately on launch.
2. `SelectionOverlay` freezes the current virtual desktop and lets the user drag a region.
3. Selected region is captured with `mss`.
4. Rounded corners are applied immediately after capture.
5. `EditorWindow` opens with toolbar + canvas.
6. Social presentation background is enabled by default in the editor preview.
7. User can annotate, switch presentation layout/color, recapture, copy, or save.
8. Closing the editor quits the app.

## Current Architecture

### `main.py`
- No tray, no hotkey, no background process.
- Qt DPI auto-scaling is disabled so capture/editor match real pixel size.
- Process DPI awareness is enabled on Windows before Qt starts.
- `assets/logo.jpg` is used as the app/window icon when running from source, cropped into a circular icon at load time.

### `core/`
- `capture.py`: selection overlay using a frozen screenshot background.
- `screenshot.py`: capture via `mss`.
- `clipboard.py`: copies PNG mime data to clipboard to preserve alpha/rounded corners when the target app supports it.
- `paths.py`: resolves bundled assets correctly for both source-run and PyInstaller builds.
- `settings.py`: local persistence for the chosen presentation background color.
  - custom background images are now restricted to local filesystem paths only; UNC/network paths are rejected
  - presentation color mode now also remembers `solid` vs `gradient`, plus the selected gradient preset

### `editor/`
- `editor_window.py`: editor shell, keyboard shortcuts, save/copy actions, minimum-width guard against toolbar overlap.
  - Copy now shows a centered `Copied` toast overlay in the preview area and fades it out over about 2 seconds.
- `canvas.py`: annotation storage, undo, render pipeline, flatten/export pipeline.
- `presentation_view.py`: wraps the live canvas in a social-style preview background.
- `toolbar.py`: compact ribbon with tools, 4-color palette, actions, and background controls.
  - Toolbar now also has a subtle icon-only X button at the far right that opens `https://x.com/solotop999` in the default browser.
- `tools/`:
  - `rect_tool.py`
  - `arrow_tool.py`
  - `label_tool.py`
  - `text_tool.py`
    - text notes now use a clearer note-card style with padded text, a soft rounded background, and better contrast in both edit mode and final render
  - `redact_tool.py`

### `processing/`
- `corners.py`: rounded corner alpha mask.
- `blur.py`: helper module, now also contains secure solid-black redact logic.
- `presentation.py`: social background composition, layout sizing, and export shadow.
  - color backgrounds now support curated linear gradient presets for preview/export parity

## Important Visual / UX Decisions
- App title is `Solotop Capture`.
- Default annotation color is green: `#19C85B`.
- Color palette is only 4 colors:
  - green
  - red
  - black
  - white
- Palette is compact and uses circular chips.
- Selected color shows a blue outer ring like a system color-picker chip.
- Rect and Arrow use thicker default strokes than before.
- Undo icon was changed to a simple system-style glyph.
- Default presentation mode is `BG on + Original + saved color`.
- `Original` keeps the screenshot's original aspect ratio and just adds fixed padding around it.
- Background now uses bundled asset artwork as a deep-blurred backdrop, then tints it with the user-picked color.
- The chosen color is still the main surface tone; the asset images act as soft texture behind it.
- V1 does not allow user-supplied background images.
- `_presentation_group` in the toolbar is intended to behave like a fixed-width menu cluster, not a responsive section.

## Redact Behavior
- Redact is no longer blur or mosaic.
- Redact now overwrites the selected region with solid black for stronger safety.
- Preview while dragging uses a plain dark rounded overlay.
- After commit, the editor displays the actual solid-black redacted result.
- Export/save uses the same solid-black result.
- Redact/export pipeline preserves alpha, so rounded corners should remain intact.

## Security / Packaging Notes
- No network modules are intentionally used by the app flow.
- One explicit user-requested exception exists: the X icon button opens the user's X profile in the default browser when clicked.
- `build.spec` now bundles the runtime assets from `assets/` needed for app icon, built-in presentation backgrounds, and the X button icon.
- `build.spec` now supports `one-file` PyInstaller output, and the Windows executable icon is taken from `assets/logo.ico` generated from `logo.jpg`.
- `build.spec` was simplified and no longer depends on tray/icon assets.

- Do not build `.exe` again unless the user explicitly asks.
- Social background generation is fully local/offline and uses Pillow only.

## Performance Optimizations Applied
- `_pil_to_qpixmap()` in `core/capture.py` and `editor/canvas.py` replaced PNG encode/decode roundtrip with direct raw bytes → `QImage.Format_RGBA8888` → `QPixmap`. ~57% faster.
- `processing/corners.py` supersampling reduced from 4x to 2x. ~54% faster.
- Rounded corner radius changed from 28 to 22 in `main.py`.
- Added missing `QPainterPath` import in `editor/tools/text_tool.py`.
- Removed unused `import io` from `core/capture.py`.

## Known Caution
- Clipboard transparency depends partly on the target app:
  - apps that support `image/png` clipboard data can preserve rounded corners
  - apps that only read legacy bitmap clipboard formats may still flatten transparency
